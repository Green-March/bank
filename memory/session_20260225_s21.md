### 提案詳細

**A: market-data-collector 新規作成**
- J-Quants 株価API (`/v1/prices/daily_quotes`) + 上場銘柄情報API (`/v1/listed/info`) からデータ取得
- PER/PBR/時価総額/EV/EBITDA倍率/配当利回り算出が可能に
- auth.py の認証基盤を流用可能
- 「財務分析レポート」→「投資分析レポート」への質的転換

**B: example_pipeline.yaml 7ステップ化**
- 現行: collect → parse → calculate → report (4ステップ)
- 目標: resolve → collect → parse → integrate → calculate → inventory → report (7ステップ)
- pipeline-runner 本体は完全実装済み。YAML定義更新のみ
- 3銘柄目以降の追加コスト大幅削減

**C: financial-reporter 終了コード修正**
- main() が sys.exit() を呼び出さず、エラー時も常に終了コード0を返すバグ
- pipeline-runner がreporter失敗を検知できない（サイレント品質劣化）

**D: comparable-analyzer peer欠損時 graceful degradation**
- peer企業の metrics.json 不在時にクラッシュ → 利用可能なデータのみで比較実行に変更
- 欠損リストを missing_peers フィールドに記録
